Running simulations using GROMACS and PLUMED at DIPC systems

For any information related with how DIPC computing system works (ATLAS), please see the corresponding web pages. Here, we will restrict ourselves to the EDR part of Atlas, which is powered by GPUs.

Serial jobs

The first thing you need to do in Atlas for using any software is to load the corresponding module

module load GROMACS/2021.3-fosscuda-2020b-PLUMED-2.7.2

Below, you can find a batch script for parallel execution

#!/bin/bash
#SBATCH --partition=regular
#SBATCH --job-name=GROMACS_job
#SBATCH --mem=200gb
#SBATCH --cpus-per-task=6
#SBATCH --nodes=2
#SBATCH --ntasks-per-node=8
#SBATCH --output=%x-%j.out
#SBATCH --error=%x-%j.err

module load GROMACS/2021.3-fosscuda-2020b-PLUMED-2.7.2

srun  gmx_mpi mdrun -ntomp ${SLURM_CPUS_PER_TASK} -s input.tpr

You can specify the number of GPUs that you are requesting making a small modification of the script 

#!/bin/bash
#SBATCH --partition=regular
#SBATCH --job-name=GROMACS_job
#SBATCH --mem=200gb
#SBATCH --cpus-per-task=1
#SBATCH --nodes=2
#SBATCH --ntasks-per-node=48
#SBATCH --gres=gpu:p40:2
#SBATCH --output=%x-%j.out
#SBATCH --error=%x-%j.err

module load GROMACS/2021.3-fosscuda-2020b-PLUMED-2.7.2

srun gmx_mpi mdrun -ntomp $SLURM_CPUS_PER_TASK -nb auto -bonded auto -pme auto -gpu_id 01 -s input.tpr

Multiple replicas

In order to run GROMACS with multiple replicas, remember to add the corresponding flags. The example below is for replica exchange for solute tempering (-hrex), which will require the existence of a plumed.dat file, which can be empty.

#!/bin/bash
#SBATCH --partition=regular
#SBATCH --job-name=GROMACS_plumed
#SBATCH --mem=64gb
#SBATCH --nodes=1
#SBATCH --cpus-per-task=1
#SBATCH --ntasks-per-node=8
#SBATCH --gres=gpu:p40:1
#SBATCH --time=00:10:00

module load GROMACS/2021.3-fosscuda-2020b-PLUMED-2.7.2

tpr="input_rest2"
out=$tpr

mpirun -np ${SLURM_NTASKS} gmx_mpi mdrun -s $tpr -ntomp 1 -v -multidir rep* -nsteps 1000 -hrex -replex 100 -deffnm $out -plumed ../plumed.dat
